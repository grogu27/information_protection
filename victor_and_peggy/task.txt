Протокол Фиата-Шамира
Для выполнения этого варианта задания необходимо разработать клиентсерверное приложение с авторизацией по протоколу Фиата-Шамира.
Открытые ключи с соответствующими логинами должны храниться в файле
(или базе данных) на сервере, клиентское приложение при этом не должно
отправлять на сервер никаких закрытых данных, закрытый ключ нигде не
хранится и используется исключительно для осуществления работы
протокола с клиентской стороны. Все открытые параметры системы
рассылаются сервером при установке соединения с клиентом.


n=pq — произведение двух больших простых чисел ,
s — секрет,
v — открытый ключ.

P: choose r
x = r^2 mod n → V

v: choose e ∈ {0,1} → P

P:
if e == 0: y = r
else: y = r*s mod n

V:
check y^2 == x * v^e mod n

Свойства протокола
✔ Полнота

Честный доказывающий всегда проходит проверку.

✔ Корректность (soundness)

Злоумышленник без секрета s может обмануть только с вероятностью  1/(2^k)


✔ Нулевое разглашение

Проверяющий не получает никакой информации о s.




login | n | v


s = 7       // секрет клиента
p = 11, q = 17
n = 187
v = s^2 % n = 49
k = 3       // число раундов
